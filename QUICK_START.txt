QUICK START (RECOMMENDED)
═════════════════════════════════════════════════════════════════════════════

1. Make the start script executable:
   $ chmod +x start.sh

2. Run the build script (builds all three services):
   $ ./start.sh

3. Start each service in separate terminals:

   TERMINAL 1 - Core Banking (Port 8082):
   $ cd system2-corebank && mvn spring-boot:run

   TERMINAL 2 - Transaction Gateway (Port 8081):
   $ cd system1-gateway && mvn spring-boot:run

   TERMINAL 3 - React Frontend (Port 3000):
   $ cd banking-ui && npm run dev

4. Open your browser and access:
   http://localhost:3000

MANUAL BUILD & RUN (ALTERNATIVE)
═════════════════════════════════════════════════════════════════════════════

System 2 (Core Banking) - Start First:
$ cd system2-corebank
$ mvn clean package
$ mvn spring-boot:run
  → Runs on http://localhost:8082
  → H2 Console: http://localhost:8082/h2-console

System 1 (Gateway) - Start Second:
$ cd system1-gateway
$ mvn clean package
$ mvn spring-boot:run
  → Runs on http://localhost:8081

Banking UI (Frontend) - Start Third:
$ cd banking-ui
$ npm install
$ npm run dev
  → Runs on http://localhost:3000


TESTING THE API (USING cURL)
═════════════════════════════════════════════════════════════════════════════

Test 1: Submit a top-up transaction
$ curl -X POST http://localhost:8081/api/transaction \
  -H "Content-Type: application/json" \
  -d '{
    "cardNumber": "4123456789012345",
    "pin": "1234",
    "amount": 100.00,
    "type": "topup"
  }'

Test 2: Check balance
$ curl -X GET http://localhost:8082/api/balance/4123456789012345

Test 3: View customer transactions
$ curl -X GET http://localhost:8082/api/transactions/customer/4123456789012345

Test 4: View all transactions (admin)
$ curl -X GET http://localhost:8082/api/transactions/all

OR use the automated test script:
$ chmod +x test-api.sh
$ ./test-api.sh

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Issue: "Connection refused" on port 8082
Solution: Start System 2 first, wait 5 seconds before starting System 1

Issue: CORS errors in browser
Solution: Check that @CrossOrigin annotations are present in controllers

Issue: PIN not working
Solution: Remember PIN is hashed with SHA-256 (1234 for card ending in 5345)

Issue: "npm: command not found"
Solution: Install Node.js from https://nodejs.org/

Issue: Maven dependency errors
Solution: Run: mvn clean install -U
